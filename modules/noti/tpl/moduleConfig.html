작동 정지. 필수로 입력되어야 할 설정값이 누락되어있습니다. {$lang->help}

정지 상태일 경우 모듈의 '푸시 서버 설정', '메세지 큐 서버 설정' 항목을 모두 입력해주세요.
작동중이여도 rabbitmq, nodejs서버가 동작중이지 않거나 연동이 제대로 되지 않을 경우 동작하지 않습니다.

푸시 서버 설정

{$lang->help}

VAPID를 이용하여 푸시 알림을 보낼 경우 사용할 Public Key를 입력합니다.

{$lang->help}

VAPID를 이용하여 푸시 알림을 보낼 경우 사용할 Private Key를 입력합니다.

{$lang->help}

VAPID 클레임에 사용될 subject를 입력합니다. 알림 전송에 참조할 이메일(혹은 사이트 주소)을 입력해주세요. 이메일 입력 권장.

{$lang->help}

보안을 위해 클라이언트 단말기의 endpoint 구독정보를 제한적으로 등록합니다.
행으로 규칙을 구분합니다.
기본값 : https://fcm.googleapis.com/, https://updates.push.services.mozilla.com/, https://bn3p.notify.windows.com/, https://sg2p.notify.windows.com/
값은 https://로 시작해야 하며 도메인(IP)의 끝은 /로 끝나야 합니다.

특별한 목적이 있지 않은 경우 이 값을 수정하지 마세요.

메세지 큐 서버 설정

{$lang->help}

메세지 큐 서버 IP를 입력합니다.

{$lang->help}

메세지 큐 서버 포트를 입력합니다.

{$lang->help}

메세지 큐 서버에서 사용할 가상호스트를 입력합니다.

{$lang->help}

메세지 큐 서버접속에 사용할 ID를 입력합니다. 메세지 큐 서버에 계정이 등록되어 있어야 합니다.

{$lang->help}

메세지 큐 서버접속에 사용할 계정의 password를 입력합니다.

{$lang->help}

nodejs 서버로 처리 대기중인 큐 데이터를 삭제합니다.
대량푸시 전송을 긴급히 중단해야 할 경우 node서버 강제종료 요청 후, 이 기능을 사용하세요.

{$lang->help}

node서버 프로세스 강제종료 시그널을 메세지 큐를 통하여 node서버로 보냅니다.
이미 전송중인 푸시작업을 무시하고 강제로 서버를 종료합니다.
대량푸시 전송을 긴급히 중단해야 할 경우 이 기능을 사용한 뒤, 큐 퍼지 기능을 사용하세요.
node 서버가 강제종료된 후, 다시시작하려면 수동으로 켜야합니다.

메세지 큐로 전달되는 node서버 강제종료 메세지(시그널)는 다른 메세지들보다 우선순위가 높습니다.

{$lang->help}

메세지 큐 서버로 접속 테스트를 시도합니다.
정상적으로 연결 테스트가 되더라도 메세지 큐 서버에 exchange, key, queue, 계정 설정이 제대로 되지 않은 경우 전송이 원할하게 되지 않을 수 있습니다.
메세지 큐 서버가 정상적으로 동작하더라도 nodejs 서버가 작동하지 않을 경우 푸시 전송이 작동하지 않습니다.
